# NiagaraDataSet.h

文件位置：`Source/Niagara/Classes/NiagaraDataSet.h`

这个文件定义了 Niagara 的**粒子数据存储**——所有粒子属性（位置、速度、颜色、年龄等）都存在这里。这是 Niagara 运行时最核心的数据结构之一。

---

## 一、SOA 数据布局

Niagara 使用 **SOA（Structure of Arrays）** 而非 AOS（Array of Structures）来存储粒子数据。

**AOS（传统方式）**：
```
[Particle0: {Pos, Vel, Color}] [Particle1: {Pos, Vel, Color}] ...
```

**SOA（Niagara 方式）**：
```
Float Buffer: [Pos.x_0, Pos.x_1, ...] [Pos.y_0, Pos.y_1, ...] [Vel.x_0, Vel.x_1, ...]
Int32 Buffer: [Age_0, Age_1, ...]
```

SOA 对 SIMD 和 GPU 更友好，因为同类型数据连续存储。

---

## 二、FNiagaraVariableLayoutInfo（第 14 行）— 变量在 Buffer 中的位置

```cpp
uint32 FloatComponentStart;  // 该变量的 float 分量在 FloatBuffer 中的起始索引
uint32 Int32ComponentStart;  // int32 分量起始索引
uint32 HalfComponentStart;   // half 分量起始索引
FNiagaraTypeLayoutInfo LayoutInfo; // 类型布局信息（来自 NiagaraTypes.h）
```

例如 `FVector`（Position）有 3 个 float 分量，`FloatComponentStart = 5` 表示它从第 5 个 float 分量列开始。

---

## 三、FNiagaraSharedObject（第 47 行）— 线程安全的共享对象

```cpp
TAtomic<int32> ReadRefCount;  // 原子引用计数
```

- `ReadRefCount > 0` → 正在被读取
- `ReadRefCount == INDEX_NONE` → 被锁定写入
- `ReadRefCount == 0` → 空闲

这是为了保证**游戏线程写入、渲染线程读取**时的线程安全。`FNiagaraDataBuffer` 继承了这个类。

## 四、FNiagaraDataBuffer（第 103 行）— ★ 单帧数据缓冲区

这是**一帧的粒子数据**。核心成员：

### CPU 数据

```cpp
TArray<uint8> FloatData;   // 所有 float 分量的连续存储
TArray<uint8> Int32Data;   // 所有 int32 分量的连续存储
TArray<uint8> HalfData;    // 所有 half 分量的连续存储
TArray<int32> IDToIndexTable; // ID → 索引的映射表
```

### GPU 数据

```cpp
FRWBuffer GPUBufferFloat;  // GPU float 缓冲区
FRWBuffer GPUBufferInt;    // GPU int32 缓冲区
FRWBuffer GPUBufferHalf;   // GPU half 缓冲区
FRWBuffer GPUIDToIndexTable; // GPU ID 映射表
uint32 GPUInstanceCountBufferOffset; // GPU 实例计数偏移
```

### 实例信息

```cpp
uint32 NumInstances;          // 当前实例（粒子）数
uint32 NumInstancesAllocated; // 已分配的容量
uint32 NumSpawnedInstances;   // 本帧新生成的粒子数
uint32 FloatStride;           // float 分量间的步长
uint32 IDAcquireTag;          // 当前 ID 分配标签
```

### 数据访问方法

```cpp
// 获取某个分量列的指针
uint8* GetComponentPtrFloat(uint32 ComponentIdx);
uint8* GetComponentPtrInt32(uint32 ComponentIdx);

// 获取某个粒子某个分量的值指针
float* GetInstancePtrFloat(uint32 ComponentIdx, uint32 InstanceIdx);
int32* GetInstancePtrInt32(uint32 ComponentIdx, uint32 InstanceIdx);
```

**内存布局示意**：
```
FloatData:
  [Component0: inst0, inst1, inst2, ...padding...]  ← FloatStride 宽
  [Component1: inst0, inst1, inst2, ...padding...]
  [Component2: inst0, inst1, inst2, ...padding...]
```

每个分量列的宽度是 `FloatStride` 字节，对齐到 `VECTOR_WIDTH_BYTES`。

### 关键方法

```cpp
void Allocate(uint32 NumInstances, bool bMaintainExisting = false);
void SwapInstances(uint32 OldIndex, uint32 NewIndex); // 交换两个粒子
void KillInstance(uint32 InstanceIdx);  // 杀死一个粒子
void CopyTo(FNiagaraDataBuffer& DestBuffer, ...); // 拷贝数据
int32 TransferInstance(FNiagaraDataBuffer& SourceBuffer, int32 InstanceIndex); // 转移粒子
void BuildRegisterTable(); // 构建寄存器表（VM 执行用）
```

## 五、FNiagaraDataSetCompiledData（第 253 行）— 数据集编译布局

```cpp
TArray<FNiagaraVariable> Variables;             // 变量列表
TArray<FNiagaraVariableLayoutInfo> VariableLayouts; // 每个变量的布局
FNiagaraDataSetID ID;                           // 数据集 ID
uint32 TotalFloatComponents;   // 总 float 分量数
uint32 TotalInt32Components;   // 总 int32 分量数
uint32 TotalHalfComponents;    // 总 half 分量数
uint32 bRequiresPersistentIDs : 1;
ENiagaraSimTarget SimTarget;   // CPU or GPU
```

`BuildLayout()` 遍历所有 Variables，使用 `FNiagaraTypeLayoutInfo` 计算每个变量的分量偏移，构建完整的 SOA 布局。

## 六、FNiagaraDataSet（第 300 行）— ★ 数据集管理器

管理多个 `FNiagaraDataBuffer`，实现双缓冲：

```cpp
FNiagaraDataBuffer* CurrentData;      // 当前帧数据（可读）
FNiagaraDataBuffer* DestinationData;  // 正在写入的数据
TArray<FNiagaraDataBuffer*, TInlineAllocator<2>> Data; // Buffer 池

// 持久 ID 管理
TArray<int32> FreeIDsTable;    // 空闲 ID 表
TArray<int32> SpawnedIDsTable; // 本帧生成的 ID
int32 NumFreeIDs;
int32 MaxUsedID;
int32 IDAcquireTag;
```

### 模拟生命周期

```cpp
// 1. 开始模拟：获取目标 Buffer
FNiagaraDataBuffer& BeginSimulate(bool bResetDestinationData = true);

// 2. 分配空间
void Allocate(int32 NumInstances, bool bMaintainExisting = false);

// 3. ... VM 或 GPU 写入 DestinationData ...

// 4. 结束模拟：DestinationData 变成 CurrentData
void EndSimulate(bool SetCurrentData = true);
```

### Buffer 池机制

```
Data 数组通常有 2-3 个 Buffer：
  Data[0] → 上一帧（可能还在被渲染线程读）
  Data[1] → 当前帧
  Data[2] → 目标（正在写入）

BeginSimulate() 在池中找一个空闲的 Buffer 作为写入目标。
EndSimulate() 把写入完成的 Buffer 标记为 CurrentData。
```

这样保证了渲染线程读取上一帧数据时，游戏线程可以安全地写入新一帧。

## 七、FNiagaraDataVariableIterator（第 454 行）

用于逐粒子遍历数据的调试迭代器。**很慢，不要在运行时使用**。按变量逐个读取所有分量并组装成 `FNiagaraVariable`。

---

## 总结

1. **SOA 布局**：粒子数据按分量类型（float/int32/half）分开存储，对 SIMD/GPU 友好
2. **FNiagaraDataBuffer** 是一帧的数据，包含 CPU 和 GPU 两套存储
3. **FNiagaraDataSet** 管理双缓冲，保证读写线程安全
4. **FNiagaraDataSetCompiledData** 描述数据集的布局（哪些变量、多少分量）
5. 持久 ID 系统通过 `FreeIDsTable` 管理 ID 的分配和回收
6. Buffer 池机制解决了游戏线程/渲染线程并行访问的问题
