# NiagaraParameterStore.h

文件位置：`Source/Niagara/Public/NiagaraParameterStore.h`

`FNiagaraParameterStore` 是 Niagara 的**参数存储中心**——模块输入、系统参数、用户参数、数据接口引用，都存在这里。它是模块间、脚本间、系统间数据传递的关键机制。

---

## 一、FNiagaraParameterStoreBinding（第 33 行）— 参数绑定

ParameterStore 之间可以建立**绑定关系**。当源 Store 的参数变化时，自动推送到目标 Store。

### 三种绑定类型

```cpp
struct FParameterBinding {
    uint16 SrcOffset;   // 源偏移
    uint16 DestOffset;  // 目标偏移
    uint16 Size;        // 数据大小
};
// → 用于普通参数数据（float、int 等）

struct FInterfaceBinding {
    uint16 SrcOffset;
    uint16 DestOffset;
};
// → 用于 DataInterface 引用

struct FUObjectBinding {
    uint16 SrcOffset;
    uint16 DestOffset;
};
// → 用于 UObject 引用
```

### 关键方法

```cpp
bool Initialize(FNiagaraParameterStore* DestStore, FNiagaraParameterStore* SrcStore);
void Tick(FNiagaraParameterStore* DestStore, FNiagaraParameterStore* SrcStore, bool bForce);
```

`Initialize` 遍历两个 Store，找到名字和类型匹配的参数，建立偏移映射。`Tick` 时只需要按偏移 memcpy，非常高效。

## 二、FNiagaraParameterStore（第 148 行）— ★ 核心参数存储

### 数据成员

```cpp
UObject* Owner;  // 拥有者

// 参数变量和偏移（排序后的）
TArray<FNiagaraVariableWithOffset> SortedParameterOffsets;

// 三种数据存储
TArray<uint8> ParameterData;                    // 普通参数的原始字节
TArray<UNiagaraDataInterface*> DataInterfaces;  // 数据接口
TArray<UObject*> UObjects;                      // UObject 引用

// 绑定关系
TArray<BindingPair> Bindings;         // 我推送数据到这些 Store
TArray<FNiagaraParameterStore*> SourceStores; // 这些 Store 推送数据给我

// 脏标记
uint32 bParametersDirty : 1;   // 参数数据��修改
uint32 bInterfacesDirty : 1;   // DI 已修改
uint32 bUObjectsDirty : 1;     // UObject 已修改
uint32 LayoutVersion;           // 布局版本号
```

### 存储方式

```
SortedParameterOffsets:
  [0] {Name="Particles.Position", Type=Vec3, Offset=0}
  [1] {Name="Particles.Velocity", Type=Vec3, Offset=12}
  [2] {Name="Particles.Color",    Type=Color, Offset=24}

ParameterData: [12 bytes Position | 12 bytes Velocity | 16 bytes Color | ...]
                ↑ Offset=0         ↑ Offset=12         ↑ Offset=24

DataInterfaces: [DI_0, DI_1, ...]  ← DataInterface 类型的参数存在这里
UObjects: [UObj_0, UObj_1, ...]     ← UObject 类型的参数存在这里
```

`FNiagaraVariableWithOffset` 中的 `Offset` 含义取决于参数类型：
- 普通参数 → `ParameterData` 中的字节偏移
- DataInterface → `DataInterfaces` 数组的索引
- UObject → `UObjects` 数组的索引

## 三、核心操作

### 添加/删除参数

```cpp
virtual bool AddParameter(const FNiagaraVariable& Param, bool bInitialize, bool bTriggerRebind, int32* OutOffset);
virtual bool RemoveParameter(const FNiagaraVariableBase& Param);
virtual void RenameParameter(const FNiagaraVariableBase& Param, FName NewName);
virtual void Empty(bool bClearBindings = true);
```

`AddParameter` 在 `ParameterData` 末尾追加空间（或在 DataInterfaces/UObjects 追加槽位），更新 `SortedParameterOffsets`，递增 `LayoutVersion`。

### 读取参数

### 写入参数

```cpp

void SetParameterData(const uint8* Data, int32 Offset, int32 Size);
```

写入后自动标记 `bParametersDirty = true`，下次 `Tick()` 时推送到绑定的 Store。

### 绑定和推送


## 四、数据推送链

在运行时，参数数据通过绑定链自动流动：

```
ExposedParameters (用户在蓝图中设置)
  ↓ Bind + Tick
InstanceParameters (SystemInstance 的参数)
  ↓ Bind + Tick
ScriptExecutionParameterStore (脚本执行时的参数)
  ↓ 被 VM 读取
```

每帧 `Tick()` 时：
1. 检查 `bParametersDirty`
2. 如果脏，遍历所有 `Bindings`
3. 对每个绑定，按 `FParameterBinding` 的偏移做 memcpy
4. 推送 DI 和 UObject 引用

这种设计让参数传递极为高效——不需要按名字查找，直接按偏移拷贝。

---

## 五、EDataInterfaceCopyMethod

```cpp
Value     → 创建新 DI 实例，用 CopyTo 同步
Reference → 直接引用源 DI（共享同一个实例）
None      → 不拷贝 DI
```

## 总结

1. `FNiagaraParameterStore` 是 Niagara 的**万能参数容器**，存储普通值、DataInterface 和 UObject
2. 参数按名字排序，通过**字节偏移**索引，避免运行时的名字查找
3. **绑定机制**实现 Store 间的自动数据推送，通过预计算的偏移映射做高效 memcpy
4. **脏标记 + Tick** 模式：修改标脏，每帧统一推送
5. `LayoutVersion` 用于检测布局变化，触发重新绑定
