# NiagaraTypes.h

文件位置：`Source/Niagara/Public/NiagaraTypes.h`

这个文件定义了 Niagara 的**类型系统**——所有变量、参数、属性的类型基础都在这里。

---

## 一、基础值类型（第 20-153 行）

Niagara 定义了自己的一套基础类型，用于 VM 和 GPU 执行：

| 类型 | 说明 |
|---|---|
| `FNiagaraFloat` | float 包装 |
| `FNiagaraInt32` | int32 包装 |
| `FNiagaraBool` | **特殊的 bool**：True = `INDEX_NONE`（0xFFFFFFFF），False = 0。这是为了兼容 VM 的 compare/select 操作 |
| `FNiagaraHalf` | 半精度浮点（uint16） |
| `FNiagaraHalfVector2/3/4` | 半精度向量 |
| `FNiagaraNumeric` | 泛型数值占位符（用于类型推导） |
| `FNiagaraMatrix` | 4x4 矩阵（4 个 FVector4） |

**重点**：`FNiagaraBool` 的 True 不是 1 而是 `0xFFFFFFFF`，这是因为 VM 使用位掩码做选择操作。

## 二、FNiagaraSpawnInfo（第 156 行）— 生成信息

```cpp
int32 Count;          // 要生成多少粒子
float InterpStartDt;  // 第一个粒子的子帧偏移时间
float IntervalDt;     // 每个粒子之间的时间间隔
int32 SpawnGroup;     // 生成组标识
```

这是 Emitter 每帧传递给 Particle Spawn 脚本的数据。`InterpStartDt` 和 `IntervalDt` 支持**插值生成**（Interpolated Spawning），让高速生成的粒子在帧内均匀分布。

---

## 三、FNiagaraID（第 179 行）— 粒子唯一标识

```cpp
int32 Index;       // 在间接表中的索引（会被复用）
int32 AcquireTag;  // 获取时的唯一标签（区分复用的 Index）
```

当开启 Persistent IDs 时，每个粒子有一个 `FNiagaraID`。`Index` 是快速访问的索引但会在粒子死亡后被回收复用，`AcquireTag` 是唯一标签，用来区分不同"世代"的同一个 Index。

---

## 四、FNiagaraTypeLayoutInfo（第 214 行）— 类型的内存布局

```cpp
TArray<uint32> FloatComponentByteOffsets;    // float 分量的字节偏移
TArray<uint32> FloatComponentRegisterOffsets; // float 分量的寄存器偏移
TArray<uint32> Int32ComponentByteOffsets;     // int32 分量
TArray<uint32> HalfComponentByteOffsets;      // half 分量
```

这是类型系统最底层的布局描述。它通过反射遍历 UStruct 的属性，把每个基础类型分量的偏移记录下来。例如 `FVector` 会产生 3 个 Float 分量。

`GenerateLayoutInfo()` 递归遍历 struct 属性，按 float/int32/uint16 分类记录偏移。这是 Niagara **SOA（Structure of Arrays）** 数据布局的基础。

## 五、执行状态枚举

### `ENiagaraExecutionState`（第 330 行）

```
Active        → 正常运行，允许生成
Inactive      → 运行但不再生成新粒子
InactiveClear → 清除已有粒子并变为 Inactive
Complete      → 完成，系统可以销毁
Disabled      → （仅 Emitter）完全禁用
```

### `ENiagaraExecutionStateSource`（第 321 行）— 状态来源优先级

```
Scalability < Internal < Owner < InternalCompletion
```

优先级从低到高。Scalability 系统可以把 Emitter 设为 Inactive，但如果 Owner 设置了 Active，Owner 的优先级更高。

## 六、参数作用域 ENiagaraParameterScope（第 482 行）

这是理解 Niagara 变量命名空间的关键：

| 作用域 | 含义 |
|---|---|
| `Input` | 模块的输入参数 |
| `User` | 暴露给组件的参数（蓝图可编辑） |
| `Engine` | 引擎提供的只读参数 |
| `Owner` | 所属组件/Actor 的参数 |
| `System` | 系统级属性（跨帧持久） |
| `Emitter` | 发射器级属性（跨帧持久） |
| `Particles` | 粒子级属性（跨帧持久） |
| `ScriptTransient` | 当前阶段的临时变量（不跨阶段） |
| `Local` | 脚本内部局部变量 |
| `Output` | 脚本的输出 |

对应编辑器中变量名的前缀：`Particles.Position`、`Emitter.Age`、`System.ExecutionState` 等。

## 七、FNiagaraTypeDefinition（第 771 行）— 核心类型定义

这是整个类型系统的核心类。每个 Niagara 变量都有一个 `FNiagaraTypeDefinition`。

```cpp
UObject* ClassStructOrEnum;  // 底层的 UStruct/UClass/UEnum
uint16 UnderlyingType;       // UT_None / UT_Class / UT_Struct / UT_Enum
```

**类型判断方法**：
- `IsDataInterface()` — 是否是数据接口（UClass 类型）
- `IsUObject()` — 是否是 UObject
- `IsEnum()` — 是否是枚举
- `IsFloatPrimitive()` — 是否是浮点系列（float/vec2/vec3/vec4/matrix/color/quat）

**预定义类型**（静态方法获取）：
```cpp
GetFloatDef()     // float
GetIntDef()       // int32
GetBoolDef()      // bool
GetVec2Def()      // FVector2D
GetVec3Def()      // FVector
GetVec4Def()      // FVector4
GetColorDef()     // FLinearColor
GetQuatDef()      // FQuat
GetMatrix4Def()   // FMatrix
GetIDDef()        // FNiagaraID
```

## 八、FNiagaraVariableMetaData（第 643 行）— 变量元数据

UI 编辑器中变量的附加信息：

- `Description` — 变量描述
- `bAdvancedDisplay` — 是否折叠显示
- `EditorSortPriority` — 排序优先级
- `EditCondition` / `VisibleCondition` — 条件显示/编辑
- `bIsStaticSwitch` — 是否是静态开关
- `Usage` — 输入/输出/局部等

---

## 总结

`NiagaraTypes.h` 的核心概念：

1. **FNiagaraTypeDefinition** 是所有变量类型的基础，包装了 UStruct/UClass/UEnum
2. **FNiagaraTypeLayoutInfo** 描述了类型如何被拆分为 float/int32/half 分量存储在 SOA 数据集中
3. **FNiagaraBool** 使用特殊的位掩码表示（True = 0xFFFFFFFF）
4. **FNiagaraSpawnInfo** 控制每帧的粒子生成数量和插值
5. **ENiagaraParameterScope** 定义了变量的命名空间层次（User/System/Emitter/Particles 等）
6. **ENiagaraExecutionState** 控制系统和发射器的生命周期状态
