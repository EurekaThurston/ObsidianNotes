# NiagaraEmitter.h

文件位置：`Source/Niagara/Classes/NiagaraEmitter.h`

`UNiagaraEmitter` �� Niagara 的**发射器资产类**。一个 System 包含多个 Emitter，每个 Emitter 负责一类粒子的生成、更新和渲染。

## 一、事件系统相关结构（第 28-177 行）

### FNiagaraEventReceiverProperties（第 28 行）


### FNiagaraEventGeneratorProperties（第 65 行）

```cpp

```

### EScriptExecutionMode（第 93 行）— 事件脚本的执行模式

```
EveryParticle    → 对每个已有粒子执行事件脚本
SpawnedParticles → 只对因该事件而新生成的粒子执行
SingleParticle   → 只对事件指定的单个粒子执行
```

## 三、继承和合并机制

Niagara Emitter 支持**继承**：

```cpp
// 编辑器专用
UNiagaraEmitter* Parent;            // 父 Emitter
UNiagaraEmitter* ParentAtLastMerge; // 上次合并时的父快照
```

- `IsSynchronizedWithParent()` — 是否与父 Emitter 同步
- `MergeChangesFromParent()` — 从父合并变更
- `DuplicateWithoutMerging()` — 复制但断开父子关系

这类似于蓝图的继承机制，子 Emitter 可以覆盖父 Emitter 的设置。

## 五、数据流

```
UNiagaraEmitter
  ├── SpawnScriptProps.Script    → Particle Spawn 逻辑
  ├── UpdateScriptProps.Script   → Particle Update 逻辑
  ├── GPUComputeScript           → GPU 合并脚本
  ├── EventHandlerScriptProps[]  → 事件响应脚本
  ├── SimulationStages[]         → 自定义模拟阶段
  ├── RendererProperties[]       → Sprite/Mesh/Ribbon 渲染器
  └── SimTarget                  → CPU or GPU
```

---

## 总结

1. `UNiagaraEmitter` 是 Emitter 的资产定义（模板），运行时会创建 `FNiagaraEmitterInstance`
2. 每个 Emitter 持有 Spawn/Update 脚本、事件处理器、模拟阶段和渲染���
3. 通过 `SimTarget` 决定 CPU 或 GPU 模拟
4. 支持**继承机制**：子 Emitter 可以从父 Emitter 合并变更
5. 事件系统允许 Emitter 之间通过事件数据集通信
