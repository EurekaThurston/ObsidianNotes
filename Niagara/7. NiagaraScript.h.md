# NiagaraScript.h

文件位置：`Source/Niagara/Classes/NiagaraScript.h`

`UNiagaraScript` 是 Niagara 的**脚本资产类**。它代表节点图编译后的可执行逻辑——你在编辑器中连的每个模块、每个阶段，最终都编译成一个 UNiagaraScript。

---

## 一、辅助枚举（第 31-77 行）

### EUnusedAttributeBehaviour（第 32 行）

脚本运行时，未被使用的属性如何处理：
```
Copy        → 拷贝上一帧的值
Zero        → 清零
None        → 不处理
MarkInvalid → 标记为无效（0xBA 填充，用于调试）
PassThrough → 不做双缓冲，直接透传
```

### ENiagaraScriptLibraryVisibility（第 65 行）

脚本在模块库中的可见性：
```
Unexposed → 默认不可见（需关闭"Library only"过滤器才显示）
Library   → 始终可见
Hidden    → 完全隐藏（软废弃）
```

## 二、FNiagaraVMExecutableDataId（第 131 行）— 编译标识

用于 DDC（Derived Data Cache）查找编译结果的唯一键：

```cpp
FGuid CompilerVersionID;            // 编译器版本
ENiagaraScriptUsage ScriptUsageType; // 脚本用途
FGuid ScriptUsageTypeID;            // 用途实例 ID
FNiagaraCompileHash BaseScriptCompileHash; // 源图的哈希

uint32 bUsesRapidIterationParams : 1;  // 是否使用快速迭代参数
uint32 bInterpolatedSpawn : 1;         // 是否插值生成
uint32 bRequiresPersistentIDs : 1;     // 是否需要持久 ID
```

相同的 `DataId` = 相同的编译输入 = 可以复用缓存的编译结果。

## 三、FNiagaraVMExecutableData（第 241 行）— ★ 编译输出

这是脚本编译后的**核心数据**，VM 执行时直接使用：

```cpp
TArray<uint8> ByteCode;             // ★ VM 字节码
TArray<uint8> OptimizedByteCode;    // 运行时优化的字节码（不序列化）
int32 NumTempRegisters;             // 临时寄存器数量
int32 NumUserPtrs;                  // 用户指针数量（DI 等）

TArray<FNiagaraVariable> Attributes; // 使用的属性列表
FNiagaraScriptDataUsageInfo DataUsage; // 是否读取属性数据

TArray<FNiagaraScriptDataInterfaceCompileInfo> DataInterfaceInfo; // DI 信息
TArray<FVMExternalFunctionBindingInfo> CalledVMExternalFunctions; // 外部函数绑定
TArray<FVMExternalFunction> CalledVMExternalFunctionBindings;     // 实际函数指针

TArray<FNiagaraDataSetID> ReadDataSets;        // 读取的数据集
TArray<FNiagaraDataSetProperties> WriteDataSets; // 写入的数据集

TArray<FNiagaraStatScope> StatScopes; // 性能统计作用域
TArray<FNiagaraDataInterfaceGPUParamInfo> DIParamInfo; // GPU DI 参数

ENiagaraScriptCompileStatus LastCompileStatus; // 编译状态
TArray<FSimulationStageMetaData> SimulationStageMetaData; // 模拟阶段元数据
```

**执行流程**：VM 读取 `ByteCode`，使用 `RegisterTable` 访问 DataSet 中的数据，通过 `CalledVMExternalFunctionBindings` 调用 DI 函数。

## 四、UNiagaraScript 核心成员

### 基本属性

```cpp
ENiagaraScriptUsage Usage;  // 脚本用途（ParticleSpawn/Update/SystemSpawn...）
FGuid UsageId;              // 用途实例 ID（同用途多个脚本时区分）

FNiagaraParameterStore RapidIterationParameters; // 快速迭代参数
```

**快速迭代参数**（RapidIterationParameters）是 Niagara 的一个重要优化：模块输入值直接存在这里，运行时可以不重编译就修改。当你在编辑器中调整模块参数时，很多情况下不需要触发重编译。

### 编译相关

```cpp
// 编译结果
FNiagaraVMExecutableData CachedScriptVM;     // ★ 缓存的 VM 可执行数据
FNiagaraVMExecutableDataId CachedScriptVMId; // 编译 ID

// GPU 着色器资源
TSharedPtr<FNiagaraShaderScript> ScriptResource; // GPU 着色器

// 编译参数缓存
TArray<UNiagaraParameterCollection*> CachedParameterCollectionReferences;
TArray<FNiagaraScriptDataInterfaceInfo> CachedDefaultDataInterfaces;
```

## 五、脚本类型判断方法

`UNiagaraScript` 提供了大量便捷的类型判断方法：

```cpp
// 实例方法
bool IsParticleSpawnScript() const;
bool IsParticleUpdateScript() const;
bool IsParticleEventScript() const;
bool IsEmitterSpawnScript() const;
bool IsEmitterUpdateScript() const;
bool IsSystemSpawnScript() const;
bool IsSystemUpdateScript() const;
bool IsModuleScript() const;
bool IsFunctionScript() const;
bool IsDynamicInputScript() const;
bool IsStandaloneScript() const;  // Module/Function/DynamicInput
bool IsParticleScript() const;    // 所有 Particle 级别的
bool IsNonParticleScript() const; // Emitter 或 System 级别的
bool IsSpawnScript() const;       // 任何 Spawn 脚本
bool IsCompilable() const;        // 是否可编译（EmitterSpawn/Update 不可独立编译）

// 静态方法（不需要实例）
static bool IsGPUScript(ENiagaraScriptUsage Usage);
static bool ConvertUsageToGroup(ENiagaraScriptUsage InUsage, ENiagaraScriptGroup& OutGroup);
```

**注意**：`EmitterSpawnScript` 和 `EmitterUpdateScript` 不可独立编译（`IsCompilable()` 返回 false），它们被合并进 SystemSpawn/SystemUpdate 脚本中编译。

## 六、编译流程

```
节点图（Source Graph）
  → RequestCompile() / RequestExternallyManagedAsyncCompile()
    → 编译器产生 FNiagaraVMExecutableData
      → SetVMCompilationResults() 缓存结果
        → CachedScriptVM.ByteCode 可被 VM 执行
```

对于 GPU 脚本：
```
节点图 → HLSL 翻译 → Shader 编译 → FNiagaraShaderScript
```

## 七、编辑器元数据

```cpp
// 模块用途位掩码（决定模块可以放在哪些阶段）
int32 ModuleUsageBitmask;

// 分类、描述、关键字
FText Category;
FText Description;
FText Keywords;

// 依赖声明
TArray<FName> ProvidedDependencies;     // 提供的依赖（如 "ProvidesNormalizedAge"）
TArray<FNiagaraModuleDependency> RequiredDependencies; // 需要的依赖

// 废弃信息
uint32 bDeprecated : 1;
FText DeprecationMessage;
UNiagaraScript* DeprecationRecommendation;
```

模块依赖系统保证了堆栈中模块的正确顺序，例如 "Normalized Age" 必须在使用它的模块之前。

---

## 总结

1. `UNiagaraScript` 是**编译后的可执行单元**，包含 VM 字节码和 GPU 着色器
2. `FNiagaraVMExecutableData` 是核心编译输出，`ByteCode` 是 VM 直接执行的指令
3. `RapidIterationParameters` 允许不重编译就修改参数值
4. EmitterSpawn/Update 脚本不独立编译，被合并到 System 脚本中
5. 模块依赖系统（`ProvidedDependencies` / `RequiredDependencies`）保证堆栈顺序
