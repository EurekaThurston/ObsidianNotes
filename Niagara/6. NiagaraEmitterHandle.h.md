# NiagaraEmitterHandle.h

文件位置：`Source/Niagara/Classes/NiagaraEmitterHandle.h`

这是 7 个文件中最小的一个，但概念很重要。`FNiagaraEmitterHandle` 是 System 对 Emitter 的**引用封装**。

---

## 为什么需要 EmitterHandle？

`UNiagaraSystem` 不直接持有 `UNiagaraEmitter*`，而是通过 `FNiagaraEmitterHandle` 间接引用。这是因为：

1. 同一个 Emitter 资产可以被多个 System 使用，但每个 System 需要**自己的副本**（Instance）
2. 每个 System 中的 Emitter 需要**独立的名字、ID 和启用状态**
3. 编辑器需要追踪 Emitter 的来源（Source）以支持合并

---

## 核心成员

```cpp
private:
    FGuid Id;                  // 唯一标识符
    FName IdName;              // 基于 GUID 的名字（用于数据集引用）
    bool bIsEnabled;           // 是否启用
    FName Name;                // 在 System 中的显示名
    UNiagaraEmitter* Instance; // ★ Emitter 的副本实例

#if WITH_EDITORONLY_DATA
    UNiagaraEmitter* Source_DEPRECATED;          // 源 Emitter（已废弃）
    UNiagaraEmitter* LastMergedSource_DEPRECATED; // 上次合并的源（已废弃）
    bool bIsolated;             // 是否被隔离（编辑器调试用）
#endif
```

**关键点**：`Instance` 是 Emitter 的**深拷贝**，不是原始资产的引用。修改 System 中的 Emitter 不会影响原始 Emitter 资产。

## 公共接口

```cpp
// 基本信息
bool IsValid() const;
FGuid GetId() const;
FName GetIdName() const;           // 用于数据集的名字（基于 GUID）
FName GetName() const;             // 显示名
void SetName(FName InName, UNiagaraSystem& InOwnerSystem); // 设置名字（确保唯一）

// 启用/禁用
bool GetIsEnabled() const;
bool SetIsEnabled(bool bInIsEnabled, UNiagaraSystem& InOwnerSystem, bool bRecompileIfChanged);

// 获取 Emitter 实例
UNiagaraEmitter* GetInstance() const;

// 获取唯一实例名（用于脚本和参数引用）
FString GetUniqueInstanceName() const;

// 编辑器
bool IsIsolated() const;     // 是否被隔离
bool NeedsRecompile() const; // 是否需要重编译
bool UsesEmitter(const UNiagaraEmitter& InEmitter) const; // 是否使用某 Emitter
```

---

## GetUniqueInstanceName 的用途

这个方法返回类似 `"Emitter_0"` 的字符串，它用于：

1. **参数命名空间**：`Emitter_0.SpawnRate` 这样的全限定参数名
2. **数据集引用**：不同 Emitter 的数据集通过这个名字区分
3. **事件系统**：跨 Emitter 的事件通过 EmitterHandle 的 ID 关联

---

## IdName 的 HACK 说明

源码中有两处 HACK 注释：

```cpp
// HACK! Data sets used to use the emitter name, but this isn't 
// guaranteed to be unique. This is a temporary hack to allow the 
// data sets to continue work with using names...
```

早期 Niagara 用 Emitter 名字来标识数据集，但名字可能重复。现在用 GUID 派生的 `IdName` 来保证唯一性，但为了兼容性保留了这个字段。

---

## 与其他类的关系

```
UNiagaraSystem
  └── TArray<FNiagaraEmitterHandle> EmitterHandles
        ├── FGuid Id           → 唯一标识
        ├── FName Name         → 显示名
        ├── bool bIsEnabled    → 是否激活
        └── UNiagaraEmitter* Instance → Emitter 副本
              ├── Scripts...
              ├── Renderers...
              └── EventHandlers...
```

运行时，`FNiagaraSystemInstance` 会根据每个 `FNiagaraEmitterHandle` 创建对应的 `FNiagaraEmitterInstance`。

---

## 总结

1. `FNiagaraEmitterHandle` 是 System → Emitter 的**间接层**
2. 它持有 Emitter 的**副本**（不是引用），保证 System 间互不影响
3. 通过 GUID 保证唯一标识，解决了名字不唯一的问题
4. 提供启用/禁用、隔离等控制
5. `GetUniqueInstanceName()` 用于参数命名空间和数据集引用
