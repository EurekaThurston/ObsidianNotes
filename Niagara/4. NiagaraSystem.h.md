# NiagaraSystem.h

文件位置：`Source/Niagara/Classes/NiagaraSystem.h`

`UNiagaraSystem` 是 Niagara 的**最顶层资产类**，对应编辑器中的一个"Niagara System"。它是容器，管理多个 Emitter 和系统级脚本。

---

## 一、辅助编译数据结构（第 26-186 行）

### FNiagaraEmitterCompiledData（第 26 行）

每个 Emitter 编译后的数据：

```cpp
TArray<FName> SpawnAttributes;            // 驱动生成的属性名列表
FNiagaraVariable EmitterSpawnIntervalVar; // 生成间隔
FNiagaraVariable EmitterAgeVar;           // Emitter 年龄
FNiagaraVariable EmitterRandomSeedVar;    // 随机种���
FNiagaraDataSetCompiledData DataSetCompiledData; // 数据集布局
```

这是编译器输出，运行时 EmitterInstance 用它来初始化。

### FNiagaraSystemCompiledData（第 111 行）

系统级编译数据，包含：
- `InstanceParamStore` — 实例参数存储
- `DataSetCompiledData` — 系统数据集的编译布局
- 各种 `ParameterDataSetBindingCollection` — Spawn/Update 阶段的参数绑定（Global、System、Owner、Emitter 四个来源）

这些绑定负责在运行时把 ParameterStore 中的值**高效拷贝**��� DataSet 中，避免逐参数查找。

### FNiagaraEmitterExecutionIndex（第 178 行）

```cpp
uint32 bStartNewOverlapGroup : 1;  // 是否需要同步点（不能和前面的并行）
uint32 EmitterIndex : 31;          // Emitter 索引
```

用于 GPU 批处理时控制 Emitter 间的依赖和并行。

## 三、关键方法

### 资产管理
- `GetEmitterHandles()` — 获取所有 Emitter 句柄

## 四、编辑器功能（WITH_EDITORONLY_DATA）

- `AddEmitterHandle()` / `RemoveEmitterHandle()` — 添加/移除 Emitter
- `bBakeOutRapidIteration` — 将快速迭代参数烘焙进编译结果（优化性能）
- `bCompressAttributes` — 压缩粒子属性（Half 精度）
- `bTrimAttributes` — 移除未使用的属性
- `RequestCompileForEmitter()` — 为特��� Emitter 请求重编译
- `bExposeToLibrary` — 是否暴露到资产库
