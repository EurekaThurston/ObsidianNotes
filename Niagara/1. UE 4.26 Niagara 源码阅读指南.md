# UE 4.26 Niagara 源码阅读指南

源码位置：`D:\UE\4_26SourceCode\UnrealEngine-4.26\Engine\Plugins\FX\Niagara`

## 模块总览

```
Source/
├── NiagaraCore/            ← 最底层，基础类型和接口定义（最小依赖）
├── NiagaraShader/          ← Shader 编译相关
├── NiagaraVertexFactories/ ← 顶点工厂（渲染管线）
├── Niagara/                ← ★ 核心运行时（最大、最重要的模块）
├── NiagaraAnimNotifies/    ← 动画通知集成（很小，可忽略）
├── NiagaraEditor/          ← 编辑器（节点图、UI）
└── NiagaraEditorWidgets/   ← 编辑器控件
```

## 推荐阅读顺序

### 第一层：理解核心数据模型（先建立全局概念）

按这个顺序读 `Classes/` 和 `Public/` 下的头文件：

| 顺序  | 文件                                      | 理解什么                        |
| --- | --------------------------------------- | --------------------------- |
| 1   | [[2. NiagaraCommon.h\|NiagaraCommon.h]] | 基础枚举、类型定义，所有地方都在用           |
| 2   | [[3. NiagaraTypes.h\|NiagaraTypes.h]]                       | Niagara 的类型系统               |
| 3   | [[4. NiagaraSystem.h\|NiagaraSystem.h]]                      | **粒子系统资产**，最顶层容器            |
| 4   | [[5. NiagaraEmitter.h\|NiagaraEmitter.h]]                      | **发射器**，System 包含多个 Emitter |
| 5   | [[6. NiagaraEmitterHandle.h\|NiagaraEmitterHandle.h]]                | System 对 Emitter 的引用封装      |
| 6   | [[7. NiagaraScript.h\|NiagaraScript.h]]                       | **脚本**（编译后的节点图逻辑）           |
| 7   | [[8. NiagaraDataSet.h\|NiagaraDataSet.h]]                      | **粒子数据存储**（SOA 结构，核心数据容器）   |
| 8   | [[9. NiagaraParameterStore.h\|NiagaraParameterStore.h]]               | **参数存储**（模块间数据传递的关键）        |

### 第二层：理解运行时执行流程

| 顺序 | 文件 | 理解什么 |
|---|---|---|
| 9 | `NiagaraComponent.h` | UE 组件，将 NiagaraSystem 放入世界 |
| 10 | `NiagaraSystemInstance.h` | **系统运行时实例**（Tick 驱动核心） |
| 11 | `NiagaraEmitterInstance.h` | **发射器运行时实例**（Spawn/Update 执行） |
| 12 | `NiagaraScriptExecutionContext.h` | **脚本执行上下文**（VM 执行的入口） |
| 13 | `NiagaraSystemSimulation.h` | 系统级模拟管理（批量 Tick） |
| 14 | `NiagaraWorldManager.h` | 世界级管理器，管所有 System 实例 |

### 第三层：理解渲染

| 顺序 | 文件 | 理解什么 |
|---|---|---|
| 15 | `NiagaraRenderer.h` | 渲染器基类 |
| 16 | `NiagaraRendererSprites.h` | Sprite 渲染（最常用） |
| 17 | `NiagaraRendererMeshes.h` | Mesh 粒子渲染 |
| 18 | `NiagaraRendererRibbons.h` | Ribbon/条带渲染 |
| 19 | `NiagaraEmitterInstanceBatcher.h` | GPU 模拟批处理 |

### 第四层：理解数据接口（扩展机制）

| 文件 | 说明 |
|---|---|
| `NiagaraDataInterface.h` | **DI 基类**，理解这个就明白整个扩展体系 |
| `NiagaraDataInterfaceCurve.h` | 曲线 DI（最简单的例子） |
| `NiagaraDataInterfaceSkeletalMesh.h` | 骨骼网格 DI（复杂但实用） |

## 核心执行链路

```
NiagaraComponent::TickComponent()
  → NiagaraSystemInstance::Tick()
    → NiagaraSystemSimulation::Tick_GameThread()
      → FNiagaraEmitterInstance::Tick()  // 每个 Emitter
        → SpawnParticles / UpdateParticles
          → NiagaraScriptExecutionContext::Execute()  // VM 执行脚本
            → 读写 NiagaraDataSet（粒子属性数据）
```

## 实用建议

1. **先读头文件（.h）**，不要急着看 .cpp。头文件里的注释和成员变量就能让你理解数据模型
2. **从 `NiagaraSystemInstance.cpp` 的 `Tick` 函数入手**调试跟踪，这是运行时最核心的入口
3. `Classes/` 下的 `NiagaraDataInterface*.h` 数量很多，不需要全看，先理解基类 `NiagaraDataInterface.h`，然后挑一个简单的（如 Curve）看看实现模式即可
4. Editor 模块可以最后看，或者不看——它主要是节点图 UI，和运行时逻辑关系不大
