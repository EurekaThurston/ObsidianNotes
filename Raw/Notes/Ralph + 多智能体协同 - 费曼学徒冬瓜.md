## 视频简介(WebFetch 提取的描述)

讲解如何让 AI 长时间自主工作。涵盖两种方案:

1. **Ralph 方案**:采用 while 循环启动新会话,通过文件系统衔接上下文
2. **多智能体方案(推荐)**:主 Agent 只协调不干活,子 Agent 各司其职,开发测试分工明确

重点包括:**任务拆分、提示词设计和经验库优化**。

---

## 视频核心论点(基于简介复述)

### 让 AI 长时高品质工作的两条路

#### 路线 A:Ralph 方案

- **机制**:外部 while 循环不断起新会话,每次新会话都是干净上下文,前后会话靠**文件系统**(如 progress.md / state.json / 任务清单文件)交接状态
- **本质**:用"文件系统当持久层"换"AI 跑长任务时上下文不爆炸"
- **代价**:每次新会话冷启动,要重新加载文件、重新建立任务感;prompt cache 完全失效

#### 路线 B:多智能体协同(视频推荐)

- **角色分层**(典型):
  - 主智能体(协调者):只拆任务、派活、收结果,不亲自干
  - 计划智能体:产 plan,不写代码
  - 代码智能体:按 plan 改代码,不重新规划
  - 测试智能体:跑测试 / 审 diff,反馈给主 agent
- **三件配套**:
  - **任务拆分**(主 agent 的核心职责)
  - **提示词设计**(每个角色一份 system prompt,定义边界 + 工具集 + 输出格式)
  - **经验库**(用文件持久化"踩过的坑 / 项目约定",每轮迭代都喂回去,防同一错误重犯)

### 实践平台

视频实操是 **Claude Code**(命令行版),靠其原生 subagent 机制(`.claude/agents/<role>.md` 之类)落地多智能体。
