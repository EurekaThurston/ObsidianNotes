# 代码检索与美术问答机器人对话(2026-04-24)

> 对话归纳,非逐字。Eureka 与 Claudian(本仓 LLM 作者身份)的一次探讨,起因是 Eureka 了解到同事使用 "grep" 概念和 Claude Code + OpenClaw 向 UE 项目工程源码提问检索成功,由此引出"给美术做代码问答机器人"的项目级 AI 落地方向。

## 问题 1:你是怎么检索代码的?同事说的 "grep" 是什么?

**Claudian 的回答要点**:

1. **检索方式是 agentic grep,不是 RAG**。三件套工具:
   - `Grep`(底层 ripgrep,字面/正则)
   - `Glob`(文件名 pattern)
   - `Read`(路径+行号)
2. **没有 embedding、没有向量库、没有预建索引**。每次问题临场 grep→命中→Read 上下文→再 grep,这就是 "agentic retrieval"
3. **优势**:
   - 零维护(代码变了下次 grep 就是新的)
   - 精确(字面匹配不会近似)
   - 可追溯(每条引用都是 `path:line`)
4. **劣势**:依赖关键词命中。你得先知道或猜到 symbol 名
5. **同事看到的机器人效果**九成就是这个机制 + persona 包装
6. **和本 wiki 方法论同源**:代码和 wiki 都是符号化系统,字面匹配比向量近似更准

## 问题 2(Eureka 提出的落地想法)

"很多美术不懂材质参数、材质宏开关、静态开关、niagara 参数的具体含义,更多靠经验。如果能检索到相关代码段并加以解释,会对项目组(尤其美术)有很大帮助。"

**Claudian 的设计建议**:

- **机器人工作流**:自然语言问题 → grep 三仓 symbol → 命中 `.ush`/`.usf` 的 `#if`、C++ 的 setter、Niagara Module 的 parameter → Read 上下文 ±30 行带注释 → 用"讲给美术听"的语气重写 + 附原始 `path:line` 证据
- **Eureka 的 vault 已经是半成品弹药库**:CLAUDE.md §9 的三 code root + `Wiki/Sources/<repo>/` 源摘要页就是给这种机器人的预消化文档。Karpathy Wiki 方法论跑起来的 compounding 效应
- **两条腿并跑**:临场 grep(覆盖长尾)+ wiki 命中(覆盖高频,更精、有人类复核)
- **架构选型**(接入层 / 代码访问 / 答案风格 / 可追溯 / 沉淀回路):详见对应读本
- **关键坑**:p4 权限、机密边界、禁止幻觉("未 grep 到证据不许作答")、美术词 ≠ 代码 symbol(需要术语桥梁)

## Demo:Dithered LOD Transition 实战

Eureka 让 Claudian 试 demo:"材质实例里的 dithered lodtransition 开关是干嘛的"

Claudian 的实际检索链条:

1. 读 `code-roots.local.md` 确认 stock 本机路径
2. 并行 grep `DitheredLODTransition`(C++ 风格)和 `DITHERED_LOD_TRANSITION|DitheredLODTransition`(shader 宏风格)
3. 跨 6 个关键文件提取证据:`MaterialInstanceBasePropertyOverrides.h`(UI)、`Material.h`(父材质)、`MaterialTemplate.ush`(shader 宏 + `ClipLODTransition`)、`LocalVertexFactory.ush`(per-instance 植被)、`DepthRendering.cpp`(stencil 优化路径)、`RendererSettings.h`(mobile CVar + 性能警告)
4. 按"美术能看到 → 底层宏 → shader 实现 → 性能代价 → 判断开关建议"的叙事结构组织答案
5. 全部附 `path:line` 证据,点击可验证

## 问题 3(Eureka 的关键追问)

Eureka 敏锐发现:"你好像本来就知道要 grep 什么——那如果是项目里自己加的功能/参数/开关,你怎么知道该搜索什么?"

**Claudian 坦白**:确实靠训练数据先验。UE 原生功能名在脑子里已有候选拼写。换成项目自定义 symbol 完全失效。

**补救策略矩阵**(按推荐度):

1. **从用户可见的锚点反推**(最好用):
   - 美术说的"某 UI 开关" → grep `LOCTEXT` / `DisplayName` / `ToolTip` 字面
   - UE 约定稳定,所有美术可见文本都走 `LOCTEXT` / `FText` / `DisplayName` meta
2. **命名约定穷举**:同时试 `bXx` / `XX_` / `USE_XX` / `r.XX` / `FXx|UXx|AXx|EXx` 等 UE 前缀
3. **从已知种子向外爬**:命中一个就能 Read ±50 行,相邻 `#include`、函数、注释都是新 grep 种子
4. **停下来问**:上面都不行时,**不猜**最重要。一句自然语言比盲 grep 10 次省事

**项目特有解法 = wiki**:

- 建 `Wiki/Concepts/Glossary/术语对照表.md`(美术/策划常说 ↔ 代码 symbol ↔ 所在模块)
- 优先 ingest 自定义 `UPROPERTY` 集中的头、自定义 shader `.usf`/`.ush`、Plugin `README`/`.uplugin`、自定义 CVar 注册点
- 半年后这张术语表 + wiki 是项目 AI 的核心资产
